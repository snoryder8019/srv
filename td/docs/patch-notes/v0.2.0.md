# v0.2.0 - Gameplay Loop + Auth

**Date:** May 1, 2026
**Status:** Live runtime + Google OAuth wired

## Added

### Gameplay
- A* pathfinding on the hex grid (`services/pathfinding.js`) with cost
  preference for `pathHexes` lane. 5 unit tests, all passing.
- Server-authoritative wave engine (`services/wave-engine.js`):
  - 20Hz tick rate, 4Hz state snapshots
  - Enemy archetypes (basic/fast/tank) with HP, speed, reward
  - Tower targeting modes (nearest/first/last/strongest/weakest)
  - Wave queueing, intermissions, win/lose detection
- Socket.IO event protocol: `run:start`, `run:place-tower`,
  `state:tick`, `enemy:*`, `tower:*`, `wave:*`
- Client-side renderers (`public/javascripts/three/entities.js`):
  - Enemy spheres tinted by HP
  - Tower platforms with GLTF (or primitive fallback if no model)
  - Tracer lines on fire events
- Demo seed (`scripts/seed-demo.js`): 1 zigzag map, 2 starter towers

### Auth
- Google OAuth via Passport, callback `/auth/google/callback`
- td has its own OAuth app in Google Console - no credentials shared
  with other services on the VM
- User model with roles (user/creator/moderator/admin)
- Sessions stored in Mongo (`sessions` collection, 30d TTL)
- Auth middleware: `requireAuth`, `requireRole(role)`, `attachUserToLocals`
- Header partial shows avatar + name when logged in, sign-in button otherwise

## Changed
- Tower model: `gltfUrl` no longer required; client renders fallback primitive
- `app.js` reorganized: trust proxy, session before passport
- `package.json` version bumped to 0.2.0
- Test runner glob fixed: `tests/unit/*.test.js`
- `DB_NAME` default changed from `towersTd` -> `td` to match cluster naming
- **PORT moved from 3700 -> 3720**. Previous choice collided with
  `/srv/mobile-meadows` (which is the rightful tenant of 3700). Apache
  vhost configs `000-towers*.conf` updated to proxy to 3720.

## Independence
- td reads ONLY from `/srv/td/.env` - no fallback to `/srv/slab/.env` or
  any other service. This was briefly attempted and reverted.
- td has its own DB cluster URL (not shared with `ps`)
- td has its own Google OAuth app
- td has its own session and JWT secrets

## Incidents (resolved)
- **Port collision with mobile-meadows** (May 1): td was incorrectly assigned
  port 3700, which is mobile-meadows' port. Briefly, requests to
  `mobilemeadows.madladslab.com` were served the Towers landing page.
  Resolved by moving td to 3720 and restarting mobile-meadows in its
  `mobile-meadows` tmux session. Both subdomains now route correctly.
- **DB credential leak prevention** (May 1): An incomplete connection string
  (username:password fragment, missing `mongodb+srv://` scheme) was pasted into
  `.env`, which surfaced parts of the credential when diagnosing the parse
  error. Operator was advised to rotate the affected Mongo password.
