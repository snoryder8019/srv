# v0.2.1 - Mobile Responsiveness

**Date:** May 1, 2026
**Status:** Cell phone playable

## Mobile-first overhaul

### Layout
- Stylesheet rebuilt mobile-first with breakpoints at 640px and 1024px
- Game/builder views: stacked layout on mobile (canvas top, controls bottom),
  side-by-side on desktop (≥1024px)
- HUD/form panel becomes a 50dvh bottom sheet on mobile, collapsible via
  `▼ HUD` toggle button so players can see more board
- Tower picker: horizontal scroll-snap row on mobile, vertical list on desktop
- Header nav wraps to a second row on phones, single row on tablets+

### Touch interactions
- `OrbitControls` configured with `TOUCH.ROTATE` (1 finger) and
  `TOUCH.DOLLY_PAN` (2 fingers) - matches phone game expectations
- Tower placement uses pointer events with tap-vs-drag detection (8px / 350ms
  thresholds) so dragging the camera no longer accidentally places a tower
- Map builder paint: same tap-vs-drag pattern
- `touch-action: none` on canvas prevents page scroll fighting with gestures
- `touch-action: manipulation` on buttons removes 300ms tap delay

### Iconography & meta
- Viewport meta now includes `viewport-fit=cover` for notch handling
- `theme-color` set to match dark background (no white iOS status bar flash)
- `apple-mobile-web-app-capable` enables PWA-style add-to-homescreen
- `format-detection: telephone=no` stops iOS from auto-linking numbers
- Inputs sized to 16px font to prevent iOS zoom-on-focus
- Safe-area insets respected on header padding-top and footer/HUD padding-bottom

### Performance on mobile
- Pixel ratio capped at 2 (avoids 3x DPR on phones killing GPU)
- WebGL context loss handler added (common on iOS under memory pressure)
- Resize handler debounced via `requestAnimationFrame` to ride out iOS
  toolbar show/hide transitions
- `dvh` units used alongside `vh` for accurate viewport height with browser chrome
- Camera `minDistance: 4`, `maxDistance: 40`, `maxPolarAngle ~87°` so phone
  users can't get lost zooming or rotating below the board

### Touch targets
- All buttons enforce 44px minimum height (WCAG)
- Tap highlight color removed for cleaner feedback
- Active-state `transform: scale(0.97)` for tactile feel without animation cost

### Orientation hint
- Phones in portrait briefly see "↻ Rotate to landscape for a better view"
  near the bottom edge for 5 seconds

## Files touched

## Not changed
- Backend, models, API routes, wave engine - mobile fixes are entirely
  client/styles
