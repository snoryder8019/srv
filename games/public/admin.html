MOD FRAMEWORK SETUP
RUST — Carbon (Preferred)
Install Carbon:
curl -sSL https://github.com/CarbonCommunity/Carbon/releases/latest/download/Carbon.Linux.Release.tar.gz | tar xz -C /srv/games/rust/

Directory Structure:
/srv/games/rust/carbon/
  plugins/       ← .cs plugin files go here
  plugins/disabled/ ← disabled plugins
  configs/       ← per-plugin JSON configs
  data/          ← plugin data storage
  logs/          ← Carbon logs

Auto-detection: When /srv/games/rust/carbon/ exists, the system uses Carbon paths. Otherwise falls back to Oxide (/srv/games/rust/oxide/plugins/).

Plugin hot-reload: Carbon auto-compiles .cs files on drop. No restart needed.
RCON plugin management: c.load PluginName / c.unload PluginName / c.reload PluginName
VALHEIM — BepInEx
Plugins: /srv/games/valheim/BepInEx/plugins/*.dll
Configs: /srv/games/valheim/BepInEx/config/
Requires restart after adding/removing plugins.
L4D2 — SourceMod
Plugins: /srv/games/l4d2/addons/sourcemod/plugins/*.smx
Configs: /srv/games/l4d2/addons/sourcemod/configs/
Hot-reload: sm plugins refresh in server console.
7DTD — Mod SDK
Mods: /srv/games/7dtd/Mods/<ModName>/ (folder-based)
Each mod has: ModInfo.xml + DLLs + assets
Requires restart after adding/removing mods.
